📋 概述

SM2 协同签名客户端是协同签名系统的用户端组件，负责管理客户端密钥分量 D1，与服务端协作完成签名和解密操作。采用 Rust 语言实现，确保内存安全和高性能。

核心职责：存储客户端密钥分量 D1，发起协同签名/解密请求，确保私钥永不完整出现。

什么是协同签名？

协同签名（Co-Signature）是一种分布式密码学技术，将私钥分割为多个分量，由不同方分别持有。签名时，各方协作完成签名计算，但任何一方都无法单独获取完整私钥。

传统签名 vs 协同签名

特性 传统签名 协同签名
私钥存储 单点存储，风险高 分片存储，安全高
签名过程 单方完成 多方协作
密钥泄露风险 高 低（需多方同时泄露）
合规性 需额外措施 天然满足监管要求

🔐 协议原理

密钥生成协议

┌──────────────┐ ┌──────────────┐ │ 客户端 │ │ 服务端 │ └──────┬───────┘ └──────┬───────┘ │ │ │ 1. 生成随机数 d1 ∈ [1, n-1] │ │ 2. 计算 P1 = d1 · G │ │ │ │─────────── 发送 P1 ──────────────────────────────>│ │ │ │ 3. 生成随机数 d2 │ 4. 计算 d2Inv = d2⁻¹ mod n │ 5. 计算 Pa = d2Inv · P1 - G │ 6. 存储 (userId, d2, d2Inv, Pa) │ │ │<────────── 返回 (P2, Pa) ─────────────────────────│ │ │ │ 7. 计算完整私钥 d = d1 · d2 - 1 │ │ 8. 存储 (d1, userId, Pa) │ │ │

协同签名协议

┌──────────────┐ ┌──────────────┐ │ 客户端 │ │ 服务端 │ └──────┬───────┘ └──────┬───────┘ │ │ │ 1. 计算消息哈希 E = SM3(M) │ │ 2. 生成随机数 k1 ∈ [1, n-1] │ │ 3. 计算 Q1 = k1 · G │ │ │ │─────────── 发送 (Q1, E) ──────────────────────────>│ │ │ │ 4. 生成随机数 k2, k3 │ 5. 计算 r = (E + x1) mod n │ 6. 计算 s2, s3 │ │ │<────────── 返回 (r, s2, s3) ───────────────────────│ │ │ │ 7. 计算 s1 = k1 · s3 - r · d1 mod n │ │ 8. 计算 s = s1 · s2 mod n │ │ 9. 输出签名 (r, s) │ │ │

数学原理

完整私钥推导：
Pa = d2Inv · P1 - G = d2⁻¹ · (d1 · G) - G = (d1 · d2⁻¹ - 1) · G
因此：d = d1 · d2 - 1 (mod n)

验证：Pa = d · G ✓

代码示例

use sm2_co_sign_core::{CoSignClient, ClientConfig};

#[tokio::main]
async fn main() -> Result<(), Box> {
    let config = ClientConfig {
        server_url: "http://127.0.0.1:7094".to_string(),
        timeout: 30,
        verify_tls: false,
    };
    
    let client = CoSignClient::new(config)?;
    
    // 用户注册
    let key_pair = client.register("alice", "password").await?;
    println!("用户ID: {}", key_pair.user_id);
    
    // 用户登录
    let session = client.login("alice", "password").await?;
    
    // 协同签名
    let signature = client.sign(b"Hello, SM2!").await?;
    println!("签名: {:02x?}", signature.r);
    
    Ok(())
}

🛡️ 安全机制

🔑 密钥分片隔离

客户端持有 D1，服务端持有 D2，完整私钥 d = D1 × D2 - 1 不在任何一方存储。

🦀 内存安全

纯 Rust 实现，所有权系统保证资源正确释放，无内存泄漏风险。

🎲 随机数安全

使用密码学安全随机数生成器，每次签名使用不同的随机数。

🔐 传输安全

敏感数据使用 Base64 编码，生产环境建议使用 HTTPS。

⚠️ 安全建议：
  • 不要明文存储私钥分量 D1
  • 生产环境必须使用 HTTPS
  • 定期更换密钥对
  • 妥善保管 .d1 文件
